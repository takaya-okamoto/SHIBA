# セキュリティポリシー

## サポート対象

SHIBA は v1 開発中です。セキュリティ修正は `main` ブランチに対して行います。

## 脆弱性の報告

脆弱性を見つけた場合は、**公開 issue を作らず**、GitHub の Security Advisory
（[Report a vulnerability](https://github.com/takaya-okamoto/SHIBA/security/advisories/new)）から
非公開で報告してください。可能であれば、再現手順・影響範囲・想定される修正案を添えてください。
一次返信は 72 時間以内を目指します。

## 設計上のセキュリティ境界

SHIBA は「**LLM は乗っ取られる前提**」で、賢さではなく構造で境界を守ります。

- **v1 は外向きの行動をしない**（メール送信・カレンダー登録なし）。外部データの自動取り込みもなし。
  よって *lethal trifecta*（私的データ × 信頼できない外部入力 × 行動能力）の3本目を構造的に持ちません。
- **アクセス境界（既定 deny）** — 登録済みオーナー以外のメッセージは記憶に触れさせません
  （起動時のワンタイムコードで登録）。
- **記憶のロンダリング防御（`source_trust`）** — 貼り付け・転送・OCR 由来の事実は `untrusted` として刻み、
  常駐記憶へ自動昇格せず、想起時に降格し、（将来の）行動の引き金にしません。
- **秘密情報の三層防御** — 抽出時「値でなく事実」、入力前の PII/secret scrub、ログ redaction。
- **テレメトリなし** — 外部送信は、あなたが設定したサービス（LLM / TiDB / Telegram）のみ。

## 運用上の注意

- `.env`（権限 0600）以外に秘密情報を置かない。`terraform.tfvars` / `*.tfstate` は gitignore 済み。
- `admin_ssh_cidr` を自分の `/32` に固定。アプリの inbound ポートは開けない（long polling）。
- セットアップで使ったトークン・キーは、稼働確認後にローテーションする。
- 記憶のオフサイトバックアップ（`memory_git_remote`）は **private** リポジトリを使う。
